# 簡易Twitterクローン「mini-twitter」 — プロジェクト概要と要件定義

## 背景

個人の学習プロジェクトとして、簡易Twitterクローンを作る。
目的は、DB設計・API設計・認証・SNS機能一式を、設計から実装までフルスタックで一気通貫に学ぶこと。
不特定多数に公開しても動作する品質を目指す。

既存プロジェクト（running-course-app）が同じリポジトリの apps/ 配下にあり、
そちらは React 19 + TypeScript 5.9 + Vite 8 + Tailwind CSS v4 + TanStack Router で構成されている。

## 決定済みの技術方針

- バックエンド: Ruby on Rails 8.0（APIモード）+ PostgreSQL 17
- フロントエンド: React 19 + TypeScript 5.9 + Vite 8 SPA（分離構成）
- API: GraphQL（graphql-ruby）— 単一エンドポイント `POST /graphql`
- 認証: jwt_sessions（Access Token + Refresh Token、Redis管理）
- リアルタイム: GraphQL Subscription（SSE transport）
- 状態管理: Apollo Client（サーバー状態）+ Zustand（クライアント状態）
- UI: HeroUI v2 + Tailwind CSS v4
- フォーム: React Hook Form + Zod

> 詳細は [tech-stack.md](./tech-stack.md)、選定理由は [tech-stack-decisions.md](./tech-stack-decisions.md) を参照。

## 確定済み: 要件定義

### プロジェクト基本情報

- アプリ名: mini-twitter
- 認証方針: 全ページログイン必須（未ログインはサインアップ/ログインのみ）
- 対応デバイス: モバイルファースト + デスクトップ
- UI言語: 日本語

### MVP機能（8つ）

#### 1. ユーザー認証

- サインアップ（ユーザー名/メール/パスワード）
- ログイン / ログアウト
- セッション維持（ブラウザ再訪問で自動ログイン）
- パスワードは不可逆ハッシュで保存

#### 2. ツイート

- テキスト投稿（最大300文字、空白のみ不可）
- 削除（本人のみ、確認ダイアログあり）
- 編集機能なし（Twitter準拠）
- 投稿日時は相対時間→絶対時間で表示

#### 3. タイムライン

- フォロータイムライン（デフォルト）と全体タイムラインをタブ切り替え
- 無限スクロール（カーソルベースページネーション、20件/ページ）

#### 4. いいね

- トグル式（1ユーザー1いいね）
- いいね数表示 + いいね済みの視覚的区別
- 自分のいいね一覧ページ

#### 5. フォロー/フォロワー

- フォロー/解除（自分自身は不可）
- フォロー中/フォロワー一覧
- カウント表示

#### 6. プロフィール

- ユーザー名、表示名（最大50文字）、自己紹介（最大200文字）、アイコン画像
- アイコン画像アップロード（最大2MB、JPEG/PNG/WebP）
- プロフィール編集は本人のみ
- ユーザーのツイート一覧表示

#### 7. 検索

- ユーザー名・表示名の部分一致検索
- ツイート本文のキーワード部分一致検索
- ページネーション

#### 8. リアルタイム更新（GraphQL Subscription）

- 新着ツイート通知バナー（「N件の新しいツイート」→クリックで読み込み）
- いいね数のリアルタイム反映

### ユーザーストーリー

#### アカウント管理

- US-1: 新規ユーザーとして、ユーザー名・メールアドレス・パスワードでアカウントを作成できる
- US-2: 登録済みユーザーとして、メールアドレスとパスワードでログインできる
- US-3: ログイン中のユーザーとして、ログアウトできる
- US-4: ログイン中のユーザーとして、ブラウザを閉じて再訪問してもログイン状態が維持される

#### ツイート

- US-5: ログイン中のユーザーとして、テキスト（最大300文字）を投稿できる
- US-6: 投稿者として、確認ダイアログで同意した上で自分のツイートを削除できる
- US-7: ログイン中のユーザーとして、ツイートの投稿日時を確認できる

#### タイムライン

- US-8: ログイン中のユーザーとして、フォロー中のユーザーのツイートだけを閲覧できる（フォロータイムライン、デフォルト表示）
- US-9: ログイン中のユーザーとして、タブを切り替えて全ユーザーのツイートを新しい順に閲覧できる（全体タイムライン）
- US-10: タイムラインを下にスクロールすると、自動的に過去のツイートが追加で読み込まれる

#### いいね

- US-11: ログイン中のユーザーとして、ツイートに「いいね」できる
- US-12: ログイン中のユーザーとして、「いいね」を取り消せる
- US-13: ログイン中のユーザーとして、各ツイートのいいね数を確認できる
- US-14: ログイン中のユーザーとして、自分がいいねしたツイートの一覧を確認できる

#### フォロー / フォロワー

- US-15: ログイン中のユーザーとして、他のユーザーをフォローできる
- US-16: ログイン中のユーザーとして、フォローを解除できる
- US-17: ログイン中のユーザーとして、あるユーザーのフォロー中/フォロワー一覧を閲覧できる
- US-18: ログイン中のユーザーとして、あるユーザーのフォロー数/フォロワー数を確認できる

#### プロフィール

- US-19: ログイン中のユーザーとして、ユーザーのプロフィールページ（ユーザー名、表示名、自己紹介、アイコン、ツイート一覧）を閲覧できる
- US-20: ログイン中のユーザーとして、自分の表示名・自己紹介・アイコン画像を編集できる
- US-21: プロフィールページで、そのユーザーのツイート一覧を確認できる

#### 検索

- US-22: ログイン中のユーザーとして、ユーザー名でユーザーを検索できる
- US-23: ログイン中のユーザーとして、キーワードでツイートを検索できる
- US-24: 検索結果が多い場合、ページ送りで続きを閲覧できる

#### リアルタイム更新

- US-25: タイムライン閲覧中に、新しいツイートが投稿されたことをリアルタイムで知ることができる
- US-26: ツイート閲覧中に、いいね数の変化がリアルタイムで反映される

### 非機能要件

#### パフォーマンス

- タイムラインの初回ロード: 1秒以内（ローカル環境）
- 無限スクロールの追加読み込み: 500ms以内
- いいね/フォローの反映: UI上は即座（楽観的更新）、サーバー同期は1秒以内

#### セキュリティ

- パスワードは不可逆ハッシュで保存
- SQLインジェクション対策
- XSS対策（ユーザー入力のエスケープ）
- CSRF対策
- 認証トークンの安全な管理（有効期限、無効化）
- レート制限（認証エンドポイント: 5回/分、投稿: 30回/分）

#### 信頼性

- バリデーションエラーはユーザーに分かりやすいメッセージで表示
- ネットワークエラー時のリトライ/エラー表示
- 楽観的更新失敗時のロールバック

#### レスポンシブデザイン

- モバイルファースト設計
- ブレークポイント: モバイル(~640px) / タブレット(641-1024px) / デスクトップ(1025px~)
- タッチ操作に適したボタンサイズ（最小44x44px）

#### テスト

- バックエンド: モデル・コントローラ・サービス層のユニットテスト
- フロントエンド: 主要コンポーネントとカスタムフックのテスト
- カバレッジ目標: バックエンド 80%以上

#### 運用

- 環境変数による設定管理（秘密情報はコードに含めない）
- ログ出力（認証失敗、エラー）
- CORS設定（フロントエンド・バックエンド分離に対応）

### ビジネスルール

- BR-1: ユーザー名は一意。英数字+アンダースコア、3~15文字
- BR-2: メールアドレスは一意
- BR-3: パスワードは8文字以上
- BR-4: ツイート本文は1~300文字（空白のみ不可）
- BR-5: 自己紹介は最大200文字
- BR-6: 表示名は最大50文字
- BR-7: 自分自身をフォロー不可
- BR-8: 同一ツイートへの重複いいね不可
- BR-9: ツイート削除は投稿者本人のみ
- BR-10: プロフィール編集は本人のみ
- BR-11: アイコン画像は最大2MB、JPEG/PNG/WebP
- BR-12: ツイート削除時は確認ダイアログ必須

### 画面一覧（7ルート）

| パス | 画面 | 認証 |
|------|------|------|
| `/signup` | サインアップ | 不要（未ログインのみ） |
| `/login` | ログイン | 不要（未ログインのみ） |
| `/` | ホーム/タイムライン（タブ: フォロー中[デフォルト] / 全体） | 必須 |
| `/compose` | ツイート投稿（モバイル専用。デスクトップはインライン） | 必須 |
| `/:username` | プロフィール（タブ: 投稿 / いいね）。いいねタブは自分のみ。プロフィール編集はモーダル | 必須 |
| `/:username/connections` | フォロー/フォロワー一覧（`?tab=followers\|following` でタブ切替） | 必須 |
| `/search` | 検索（タブ: ユーザー / ツイート） | 必須 |

- レイアウト: デスクトップは左サイドバー+メインの2カラム（右サイドバーはMVP後）、モバイルはトップバー+メイン+ボトムナビ+FAB

### MVPスコープ外

- ツイートへの画像添付（テキストのみ。ただしプロフィールアイコンのアップロードはMVPに含む）
- リプライ / リツイート / 引用ツイート
- DM（ダイレクトメッセージ）
- 通知機能
- メール送信（パスワードリセット等）
- 管理者機能
- 多言語対応（日本語のみ）

### 将来の拡張候補（MVP後）

1. 画像付きツイート
2. リプライ（スレッド表示）
3. リツイート / 引用ツイート
4. 通知機能（いいね・フォロー通知）
5. DM
6. ハッシュタグ
7. ブックマーク
8. 管理者機能（ユーザーBAN、ツイート削除）
9. パスワードリセット（メール送信）
10. OAuth連携（Google / GitHub ログイン）
